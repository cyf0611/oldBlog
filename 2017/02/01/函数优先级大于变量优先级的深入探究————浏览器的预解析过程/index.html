###先由一道小题进入本文

var foo;
function foo(){}
console.log(foo);

结果是函数体function foo(){}
接着下面一道题：

结果也是函数体function foo(){}

所有就有很多人说，函数声明的优先级大于变量声明的优先级。那么，为什么呢？这就要从浏览器的预解析说起了。

##预解析流程

  1. 搜寻预解析关键字
  • 寻找var关键字
  • 寻找function关键字
  1. 执行预解析
  • 先应用var关键字声明的标识符，使这些标识符有定义
    • 标识符有定以后，使用这项标识符就不会报错了
    • 但因为没有赋值，因此其值为undefined
  • 至此标识符中保存了函数的引用

###几个需要注意的细节

  • var 关键字对同一个标识符重复使用时，除第一次有效外，其他均做忽略处理。
  • 预解析时先处理变量声明，再处理函数声明
  • 不要纠结谁的优先级高，这些只是表面现象
  • 懂得了预解析流程，一切都是浮云
